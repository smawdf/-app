# 高糖小食 Design

## 设计场景

两个人在家里、厨房或沙发上用手机快速决定一顿饭，环境通常明亮而放松；界面应像共同维护的小饭桌，温暖但不幼稚，清晰而不具备商业后台感。

## 设计原则

1. 任务优先：点菜、接单、编辑菜单等动作必须一眼可见。
2. 温暖克制：用暖白、莓红和柔粉建立关系感，不依赖阴影和装饰堆叠。
3. 双人语义：头像、角色、通知和文案始终体现“我们”，但不牺牲操作清晰度。
4. 单店一致：我的店铺、点菜页和订单页必须读取并展示同一套店铺资料和图片。
5. 响应式稳定：组件尺寸由内容、约束和断点决定，不能因文字或加载状态产生跳动。

## 色彩系统

| Token | 色值 | 用途 |
| --- | --- | --- |
| Background | `#FEF8F2` | 页面暖白背景 |
| Surface | `#FFFCF8` | 卡片、弹窗和表单容器 |
| OnSurface | `#1D1B18` | 主文字 |
| OnSurfaceVariant | `#524346` | 次级文字 |
| Primary | `#894C5C` | 主操作、选中状态、品牌强调 |
| PrimaryContainer | `#F4A7B9` | 柔粉强调面 |
| Secondary | `#78555E` | 次级操作和图标 |
| SecondaryContainer | `#FFD1DC` | 浅粉提示和徽标 |
| Tertiary | `#8B4E38` | 食物、厨房语义强调 |
| TertiaryContainer | `#F8A98E` | 暖橘辅助面 |
| OutlineVariant | `#D6C1C5` | 低对比边框 |
| Error | `#E54848` | 错误和破坏性操作 |

颜色策略是 Restrained：暖色中性色占主要面积，莓红用于操作和状态，粉色与陶土色只承担辅助语义。禁止用大面积高饱和粉色压过内容。

## 字体与层级

- 使用 Android 系统无衬线字体，不引入装饰字体。
- 主页面标题通常为 24sp，紧凑面板标题为 18-21sp。
- 正文使用 14-16sp，辅助信息使用 12-13sp。
- 标题优先使用 Bold 或 Black；正文使用 Normal 或 Medium。
- 中文行高保持字号的约 1.35-1.55 倍，按钮文字不得截断。
- 不使用随屏幕宽度连续缩放的字体，也不使用负字距。

## 间距与布局

- 基础间距单位为 4dp，常用间距为 8、12、16、20、24dp。
- 页面横向边距在手机上通常为 16-20dp，宽屏内容设置最大宽度。
- 手机优先保持单列；平板通过最大宽度、双列或更宽内容区利用空间。
- 固定格式元素必须使用 `size`、`heightIn`、`widthIn`、`aspectRatio` 或稳定网格约束。
- 主要操作必须避开系统导航区和悬浮底部导航，列表保留足够底部内容边距。

## 圆角与边框

- 重复内容卡片通常使用 14-22dp 圆角。
- 通用弹窗使用 24-28dp 圆角。
- 胶囊按钮和状态标签使用 999dp 圆角。
- 卡片默认使用 1dp 低对比暖色边框。
- 不使用无意义的粗描边、嵌套卡片或多层外框。

## 阴影与玻璃效果

- 默认 `shadowElevation = 0.dp`，通过表面色和边框建立层级。
- 毛玻璃只用于短暂的身份切换或通知反馈：背景内容模糊，前景表面保持接近不透明。
- 不把半透明卡片当作全局风格，不允许底层文字直接穿透影响可读性。

## 组件规范

- 主按钮：品牌主色、胶囊形、明确动词，必须有禁用和加载状态。
- 图标按钮：优先使用 Material/Lucide 等熟悉图标，并提供内容描述。
- 表单：统一暖白容器、14dp 左右圆角、明确错误和字数状态。
- 弹窗：标题、内容和底部操作有稳定间距；长内容可滚动并设置最大高度。
- 图片选择：统一使用“从相册选择”和“拍照上传”，头像、店铺、菜品和甜蜜时刻共用同一组件。
- 删除和解绑：使用错误色，并提供二次确认。
- 提示反馈：短、可读、不遮挡导航和后续操作。

## 页面方向

- 登录注册：品牌小狗插画是首屏识别信号，表单保持集中和安静。
- 情侣首页：左侧当前用户、中间爱心、右侧伴侣；关系信息高于快捷入口。
- 点菜：店铺信息紧凑，分类栏与菜品列表服务快速扫描和反复加购。
- 发现：搜索结果出现后隐藏推荐标签，结果区独立滚动。
- 我的店铺：封面、名称、公告只保留一个统一编辑入口，信息区紧凑。
- 订单：卡片高度克制，角色只看到自己有权限执行的状态按钮。
- 纪念日：突出日期和共同记录，甜蜜时刻默认菜品图，也可换成共同照片。
- 我的：个人资料、绑定、糖糖币和设置入口保持清晰分组。

## 图片与品牌

- App 图标和登录插画使用小金毛与马尔济斯，小金毛戴厨师帽。
- 店铺与菜品优先展示真实内容图，缺失时使用项目内明确的暖色兜底图。
- 店铺管理与点菜页必须使用同一封面字段和同一默认封面。
- 菜品外部图片优先下厨房结果，缺图时使用 Bing 兜底；禁止恢复 `hoto.cn` 和 `res.hoto.cn`。
- 用户上传的图片统一保持原始比例，不裁切、不拉伸；容器允许出现少量留白，不能因为图片比例改变页面布局。

## 动效

- 动画统一使用 `CozyMotion` token。
- 常规状态变化控制在 150-250ms，使用平滑减速曲线。
- 动效只解释状态变化、按压反馈、加入购物车和提示出现，不做装饰性编排。
- 禁止弹跳、弹簧式夸张运动和页面加载表演。

## 可访问性与适配

- 文字与背景保持足够对比度，重要信息不能只靠颜色表达。
- 所有图标操作提供中文内容描述，触控区域目标不小于 44dp。
- 小屏和键盘弹出场景下，弹窗内容可滚动，主操作仍可到达。
- 长昵称、店铺名、公告和菜名使用换行或省略策略，不与相邻组件重叠。
- 横屏和平板使用内容最大宽度，避免组件无边界拉伸。

## 禁止项

- 多商家外卖平台视觉、商家排行榜、骑手和促销横幅。
- 大面积渐变、发光球、装饰性阴影和嵌套卡片。
- 同一操作出现多个不同入口或不同按钮样式。
- 使用可见说明文字解释界面如何使用，能用结构和标准控件表达时应直接表达。
- 修改底部导航结构或视觉，除非产品导航目标先发生变化。
