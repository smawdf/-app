# 菜单添加、发现搜索与会员文案清理设计

日期：2026-06-30

## 背景

当前 OrderDisk 已复刻出五个底部主 Tab：`首页 / 菜单 / 发现 / 订单 / 我的`。最新页面里仍有三个需要收口的问题：

- `菜单` 页的添加入口不够明确，添加弹层会被 `排序`、`分类管理`、空状态等入口触发。
- `发现` 页目前是静态任务入口，不能直接搜索菜品。
- App 内仍出现 `会员`、`VIP`、`饭票` 相关文案和卡片，用户要求全部移除。

## 目标

1. `菜单` 页只通过独立的加号按钮打开添加功能。
2. `发现` 页加入菜品搜索能力，复用现有搜索链路和旧菜谱接口类。
3. 全局去掉可见的 `会员`、`VIP`、`饭票` 字样和对应功能卡片。
4. 保持现有五 Tab 架构、Compose 风格和粉色轻量视觉方向。

## 非目标

- 不新建复杂会员替代体系。
- 不重做完整搜索页导航。
- 不更换网络 API、密钥读取方式或 Supabase 架构。
- 不调整与本需求无关的订单、结算、购物车核心业务逻辑。

## 方案选择

采用 A 方案：轻量接线。

- 菜单页：在顶部操作区增加明确的 `+` 圆形按钮；`排序` 与 `分类管理` 只保留视觉/未来入口，不触发添加弹层。
- 发现页：在发现页顶部增加搜索框与结果区，搜索时复用已有本地菜品、商家菜单、外部图源能力，并接入 `TianRecipeRemoteDataSource` 作为旧菜谱接口来源。
- 会员清理：删除或改写所有可见 `会员`、`VIP`、`饭票` 文案，移除我的页会员卡与饭票卡，预览文件和 source-scan 测试同步更新。

## 页面设计

### 菜单页

顶部结构调整为：

- 标题 `菜单`
- 搜索占位仍保留
- 小程序胶囊保留
- 次行显示 `共 X 个菜品`
- `排序`、`分类管理`、`+` 三个独立控件

交互规则：

- 点击 `+`：打开添加方式底部弹层。
- 点击 `手动添加`：关闭弹层并进入现有菜品编辑对话框。
- 点击 `广场偷菜`、`克隆厨房`：当前保持关闭弹层，不新增未完成业务。
- 点击空状态：不再打开添加弹层，避免误触。

### 发现页

发现页改为“发现 + 菜品搜索”的混合页：

- 顶部标题 `发现`
- 搜索框占位：`搜索菜品、做法、食材`
- 搜索为空时展示原任务入口卡片。
- 搜索中展示加载提示。
- 搜索有结果时展示菜品结果列表，包含来源标签：
  - `我的菜品` / `本地菜品`
  - `商家菜单`
  - `外部图源`
  - `天行菜谱`
- 搜索失败时展示温和错误提示，不阻断页面。

### 我的页

保留个人中心能力，移除会员/饭票：

- 用户头像与昵称保留。
- 原会员提示条移除。
- 原 `解锁会员` 卡移除。
- 原 `500 饭票` 卡移除。
- 列表调整为普通功能入口：
  - `订单通知设置`
  - `邀请小伙伴`
  - `厨房设置`
  - `浏览历史`
  - `新手教程`
  - `联系客服`
  - 可保留或复用现有登录/退出回调，但文案不出现会员或饭票。

### 首页

移除顶部 `VIP` 胶囊；不替换为其他付费或会员概念。

## 数据流

### 发现页搜索

新增或扩展发现页 ViewModel：

1. 输入框更新 `query`。
2. debounce 后触发搜索。
3. 并发或顺序读取：
   - `DishRepository.searchDishes(query)`
   - `MenuRepository.searchMenuItems(query)`
   - `ExternalDishImageSource.search(query)`
   - `TianRecipeRemoteDataSource.searchRecipes(query)`
4. 归一成发现页展示模型，例如 `DiscoverDishSearchItem`。
5. UI 根据 `idle / loading / results / empty / error` 状态渲染。

优先使用现有模型转换逻辑，避免复制过多搜索页 UI 细节。

## 错误处理

- API key 缺失：天行结果为空，不显示错误。
- 网络异常：保留本地和菜单结果，同时展示 `部分网络结果暂不可用`。
- 搜索为空：展示发现任务入口，不请求网络。
- 结果为空：展示 `没有找到相关菜品`。

## 测试

新增或更新 source-scan / unit 测试：

- 菜单页源码测试：存在加号入口；`排序`、`分类管理` 不再连接添加弹层；添加弹层文案仍存在。
- 发现页源码测试：存在搜索框、搜索结果状态、天行菜谱来源文案或 ViewModel 依赖。
- 我的页源码测试：不再包含 `会员`、`饭票`、`VIP`；保留 `厨房设置`、`新手教程`、`联系客服`。
- 首页源码测试：不再包含 `VIP`。
- 全局可见中文测试或 grep 检查：主源码和预览文件不包含 `会员`、`饭票`、`VIP`。

最终验证：

- `rtk .\gradlew.bat assembleDebug --rerun-tasks`
- `rtk git diff --check`
- `rtk git diff --cached --check`

## 风险

- 当前项目已有大量暂存改动，本轮实现必须只在相关文件内继续修改，不能重置或回滚现有变更。
- 发现页复用搜索能力时，要避免把完整搜索页 UI 直接塞入发现页导致页面过重。
- 旧菜谱接口无 key 或网络失败时，应保持页面可用。

## 验收标准

- 菜单页只有 `+` 按钮打开添加方式。
- 发现页能输入菜品关键词并展示已有接口返回或本地搜索结果。
- App 源码可见文案、测试和预览中不再出现 `会员`、`饭票`、`VIP`。
- 单元测试和 Debug APK 构建通过。
