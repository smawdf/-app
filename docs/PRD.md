# 今天吃什么？— 产品需求文档

> 版本：v1.2.0 | 更新：2026-05-08

## 产品概述

二人专属点菜与菜谱管理 Android App。解决"今天吃什么"的日常决策难题，提供菜谱搜索、随机推荐、双人协同点餐、菜品管理等功能。

**技术栈：** Jetpack Compose + Material 3 + Koin + Retrofit + Moshi + Supabase + Coil

---

## 1. 用户认证

### 1.1 注册流程（三步向导）

| 步骤 | 内容 | 校验 |
|------|------|------|
| 账号 | 邮箱 + 密码 + 确认密码 | 密码 ≥ 6 位，两次一致 |
| 个人资料 | 头像（拍照/相册/URL）+ 昵称（≤ 12 字） | - |
| 配对 | 生成配对码 / 输入对方配对码 / 跳过 | 6 位字母数字 |

注册成功 → 调用 Supabase Auth API → 同步云端数据 → 跳转首页

### 1.2 登录

- 邮箱 + 密码登录
- 支持切换至注册
- 登录成功 → 同步云端菜品/个人资料/点餐记录
- 记住邮箱自动填入

### 1.3 登出

- 个人中心 → 退出登录（确认弹窗）
- 清除 SessionManager 全部状态 → 回到引导页

## 3. 菜谱搜索

### 3.1 搜索输入

- 输入框搜索菜品/食材名称
- 300ms 防抖自动搜索
- 搜索中显示 loading 动画

### 3.2 数据源

五源并行搜索：

| 数据源 | 说明 |
|------|------|
| 我的菜单 | 用户自建菜品 |
| 本地数据库 | 内置菜谱 + 历史缓存 |
| 聚合数据 (Juhe) | POST，form 编码 |
| 天行数据 (TianAPI) | GET，query 参数 |
| 极速数据 (JisuAPI) | GET，含图片 + 步骤图 |

### 3.3 结果展示

- **Tab 切换**：全部 | 我的菜单 | 本地 | 聚合数据 | 天行数据 | 极速数据
- Tab 按全量结果动态生成，切换不消失
- 结果卡片显示：图片/emoji、菜名、烹饪时间、难度、数据源标签
- 点击跳转 DishDetailScreen
- 所有菜品均可加入心愿单

### 3.4 错误处理

- 各 API 错误独立提示（不阻断其他源的结果展示）
- API 未配置 Key 时静默跳过

## 4. 随机推荐（摇一摇）

### 4.1 筛选条件

- 分类（可选，文本输入）
- 最大烹饪时间（分钟，数字输入）
- 难度（⭐ 1-5，点击选择）

### 4.2 动画与交互

- 同心圆旋转动画（1080 度）
- 旋转中 emoji 变化
- 选中结果后脉冲动画

### 4.3 随机逻辑

- 先从本地数据库筛选（满足条件的所有菜品）
- 本地 < 20 条时自动调用聚合数据 API 补充
- SharedPreferences 记录已推荐 ID → 不重复推荐
- 全部展示过后自动重置

### 4.4 结果展示

- 主推荐结果卡片（可点击查看详情）
- 备选列表（最多 4 个）
- 推荐次数统计

## 5. 双人协同点餐

### 5.1 开始点餐

- 选择餐次：早餐 / 午餐 / 晚餐 / 夜宵
- 每餐有 emoji 图标和描述文字

### 5.2 菜品选择

- 搜索栏过滤菜品
- 2 列网格展示可选菜品
- "我的选择" 面板：已选菜品列表 + 已提交状态
- "对方选择" 面板：实时显示对方已选菜品
- 点击菜品添加至我的选择，已选菜品高亮显示
- 至少选 1 道菜才能提交

### 5.3 结果页

- 庆祝标题 + 总菜数 + 预估总时间
- 我的选择 vs 对方选择（卡片并排对比）
- 统计摘要：各自菜数、品类数、总时间、人均、配对评分
- "确认并写入历史" 按钮 → 保存至 Supabase → 返回首页

## 6. 菜品管理

### 6.1 菜品库

- 搜索过滤
- 来源筛选（全部/自建/收藏）
- 分类筛选（动态从数据中提取）
- 2 列网格卡片展示
- 长按自建菜品可删除（确认弹窗）

### 6.2 添加/编辑菜品

| 字段 | 说明 |
|------|------|
| 展示图 | 拍照 / 相册选择 / URL 输入 |
| 菜品名称 | 必填 |
| 分类 | 下拉选择：中餐、西餐、甜品、饮品、日料、韩餐、东南亚 |
| 难度 | ⭐ 1-5 |
| 烹饪时间 | 分钟数 |
| 谁爱吃 | 我 / 对方 FilterChip |
| 食材清单 | 逐条添加 + 删除 |
| 制作步骤 | 逐条添加/编辑/删除，自动编号 |
| 备注 | 多行文本 |

- 图片自动压缩（≤ 800px 宽，JPEG 85%）并上传 Supabase Storage
- **后台异步上传**：保存即时返回，图片后台上传，上传失败保留本地图片
- 保存中按钮禁用防连击
- 无 URL 输入，仅拍照/相册

### 6.3 菜品详情

- 大图展示（240dp）
- 编辑按钮（仅自建菜品，右上角）
- 菜名 + 分类 + 来源 + 创建时间
- 烹饪时间 + 难度标签
- 食材清单（2 列布局）
- 制作步骤（编号 + 描述 + 提示，带步骤图）
- 备注卡片
- "加入心愿单" 按钮（所有菜品均可收藏）

## 11. 数据架构

### 11.1 云端（Supabase）

| 模块 | 表/Bucket |
|------|-----------|
| 认证 | Auth (email + password) |
| 菜品 | `dishes` 表 |
| 个人资料 | `profiles` 表 |
| 点餐记录 | `meals` 表 |
| 图片存储 | `dish-images` bucket |

### 11.2 本地持久化

| 数据 | 方式 |
|------|------|
| 菜品缓存 | `dishes_cloud.json` (filesDir) |
| 登录状态 | SharedPreferences |
| 个人资料 | SharedPreferences (即时加载) |
| 随机推荐历史 | SharedPreferences (已推荐 ID 集合) |
| 内置菜谱 | `assets/recipes.json` (83KB) |

### 11.3 API 集成

| API | Base URL | Key 位置 |
|-----|----------|----------|
| 聚合数据 | `apis.juhe.cn` | `ApiConfig.JUHE_API_KEY` |
| 天行数据 | `apis.tianapi.com` | `ApiConfig.TIAN_API_KEY` |
| 极速数据 | `api.jisuapi.com` | `ApiConfig.JISU_API_KEY` |

## 12. 设计规范

- **风格**：清简日常 — 米白 + 深棕 + 淡绿
- **字体**：Material 3 默认
- **圆角**：卡片 12-16dp，按钮 10-20dp
- **动画**：页面切换 300ms 滑动渐入，底部 Tab 使用 crossfade
- **图标**：emoji 为主，Material Icons 为辅
