# 高糖小食 Product

## Product Register

`product`

## 一句话定位

高糖小食是一款只服务一对情侣的 Android 点菜 App。它把“今天吃什么”变成两个人共同维护的小店、点菜、做饭和记录回忆的完整流程。

## 使用者

- 一对已经绑定或准备绑定的情侣。
- “饲养员”负责店铺、菜单、接单和完成订单。
- “吃货”负责浏览菜单、点菜、结算和提交订单。
- 同一个账号允许在多台设备登录；当前版本不依赖邮箱或短信验证完成日常登录。

## 产品目标

1. 让两个人用最少步骤完成一次点菜。
2. 让店铺、菜单、订单和关系资料在双方设备间保持一致。
3. 让完成的订单自然沉淀为共同吃饭记录，而不是一次性交易。
4. 保持私密、轻量和有生活感，不引入商业外卖平台的复杂度。

## 核心流程

```text
登录/注册 -> 情侣首页 -> 点菜 -> 购物车 -> 结算 -> 订单
```

扩充菜单流程：

```text
发现 -> 搜索中文菜品 -> 加入我的店铺 -> 饲养员编辑菜品
```

关系流程：

```text
选择角色 -> 生成/输入邀请码 -> 绑定 -> 双方资料与店铺同步 -> 可主动解绑
```

## 主导航

- 首页：关系状态、双方头像、当前角色、订单提醒和快捷入口。
- 点菜：单店分类菜单、菜品详情、购物车和结算入口。
- 发现：搜索中文菜品，并加入我的店铺。
- 订单：待确认、准备中和已完成订单。
- 我的：个人资料、情侣绑定、糖糖币、店铺入口和版本信息。

## 角色规则

- 当前用户始终显示在首页关系区左侧，伴侣显示在右侧。
- 未绑定时可选择饲养员或吃货；绑定后角色锁定，解绑后才能重新选择。
- 只有吃货可以加购、结算和提交订单。
- 只有饲养员可以维护店铺、确认接单和完成订单。
- 糖糖币余额属于吃货钱包；饲养员可为伴侣增加糖糖币。

## 主要能力

- 账号密码登录和分步注册。
- 头像、昵称、角色和情侣资料。
- 邀请码绑定、绑定成功通知、解绑确认和对方通知。
- 单店店铺资料、分类、菜品、价格、库存和上下架。
- 中文菜谱本地搜索、下厨房实时搜索和 Bing 图片兜底。
- 购物车、地址、结算、订单状态和订单通知。
- 糖糖币余额与流水。
- 纪念日和甜蜜时刻，自定义照片默认回退到菜品图。
- 相册与拍照两种图片上传方式。

## 数据策略

- Supabase 保存账号、情侣关系、个人资料、店铺、菜单、订单、糖糖币、纪念日、偏好和图片 URL。
- Room 提供本地可用性、订单缓存和升级迁移；当前数据库版本为 13。
- 购物车只保存在本机，不上传云端。
- 图片上传到 Supabase Storage，并按情侣或账号范围组织目录。
- 云端同步失败不应破坏本地主要流程，应保留可恢复状态并记录客户端错误。

## 非目标

- 不做多商家、附近商家、骑手配送或平台补贴。
- 不做公开社交、排行榜和陌生人内容流。
- 不做真实货币充值、支付或商业结算。
- 不为应用商店运营增加广告、增长弹窗或营销首页。

## 首版发布标准

- 两个账号可以完成绑定、同步资料和解除绑定。
- 吃货可以提交订单，饲养员可以接单并完成订单。
- 店铺和菜单在双方设备保持一致，购物车保持本地独立。
- 图片缺失时有稳定兜底，不使用 `hoto.cn` 或 `res.hoto.cn`。
- 小屏手机与平板上不出现文字遮挡、弹窗溢出或底部导航覆盖主操作。
- Debug 与 Release 构建均通过，Release 使用本地私有签名。

## 事实源

产品行为以当前可编译源码、根目录 `AGENTS.md`、本文件和 `DESIGN.md` 为准。旧文档中的随机选菜、多商家外卖和单设备登录描述均不再作为当前产品目标。
